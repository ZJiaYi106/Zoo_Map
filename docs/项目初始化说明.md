# 微信小程序项目初始化说明

## 一、使用微信开发者工具创建 / 打开项目

1. 安装 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)（稳定版即可）。
2. 打开微信开发者工具 → **导入项目** 或 **新建项目**。
3. **目录**：选择本仓库根目录（包含 `app.json`、`project.config.json` 的文件夹，例如 `LittleApp`）。
4. **AppID**：
   - 正式开发可使用已注册小程序的 AppID；
   - 本地学习与界面调试可选择 **测试号**（或使用「游客模式 / 测试号」入口，以当前工具版本界面为准）。
5. **后端服务**：若暂无后端，可先勾选详情 → **本地设置** → **不校验合法域名、web-view（含本地模拟器）**，便于请求 `http://127.0.0.1:8000`。
6. **云服务**：创建项目时 **不要** 勾选「微信云开发 / 云开发」等云服务（本项目使用自建 FastAPI + MySQL + Redis）。

## 二、在 Cursor 中打开项目

1. 打开 Cursor → **File → Open Folder**。
2. 选择与本仓库相同的目录（与微信开发者工具中的项目目录一致）。
3. 在 Cursor 中编辑 `pages/`、`app.js` 等文件保存后，微信开发者工具会自动刷新（或手动编译）。

## 三、预览与真机调试

- 模拟器：开发者工具左侧模拟器直接预览。
- 真机：工具栏 **预览** 扫码；首次需在手机上打开「开发版小程序」并信任调试。

## 四、与后端联调

1. 启动 FastAPI（见根目录 `README.md`）。
2. 修改 `app.js` 中 `globalData.apiBase` 为电脑局域网 IP（真机调试时不能用 `127.0.0.1` 指手机自身，应填电脑的 IP，如 `http://192.168.1.100:8000`）。
3. 在微信公众平台配置 **request 合法域名**（上线必需）；本地调试依赖「不校验合法域名」。

---

按上述步骤即可保证代码在工具中 **正常预览、调试**，并与后续 Gitee 开源仓库结构一致。
