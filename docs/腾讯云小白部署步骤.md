# 腾讯云部署：小白一步步操作（刚买完域名）

按顺序做即可。某一步做不了（例如备案还在审核），就**先做能做的**，标了「可并行」的可以同时进行。

---

## 先搞懂三件事

1. **域名**：你刚买的是「网址名字」，例如 `xxx.com`。别人输入它才能找到你。  
2. **服务器**：一台**一直开着的电脑**（在云里），用来跑你的 **API 程序**（本项目里的 `backend`）。**光有域名不够，必须有一台服务器。**  
3. **备案**：如果你买的是**中国大陆**机房的云服务器，用域名访问网站前，一般要先做 **ICP 备案**（免费，但要几天到几周）。**没备案完之前，不能用域名正常访问网站**（这是政策要求，不是程序 bug）。

下面步骤里会反复用到这三点。

---

## 第一步：登录腾讯云，找到你的域名

1. 打开 [腾讯云控制台](https://console.cloud.tencent.com/)。  
2. 搜索或进入 **「域名注册」** / **「DNSPod」**，确认你买的域名已经在列表里。  
3. 先**不用改解析**，等第四步（买完服务器有 IP 再改）。

---

## 第二步：买一台云服务器（还没有就买）

1. 在腾讯云搜 **「轻量应用服务器」**（对新手简单）或 **「云服务器 CVM」**。  
2. **下单页「一键配置域名解析」**：**不强制勾选**。勾选＝自动帮你加 DNS 解析；不勾选＝买完服务器后，自己在 **DNSPod** 里手动添加 **A 记录**（如主机记录 `api` → 服务器公网 IP），与文档第四步一致。若不确定一键会解析到哪个主机名，建议**不勾选**，手动更稳。  
3. **创建方式怎么选（重要）**  
   - **推荐（与本文档命令一致）**：选 **「基于操作系统镜像」** → **Ubuntu 22.04 LTS**。不要选 WordPress、MCP、Dify 等应用模板，那些不是跑本仓库 FastAPI 用的。  
   - **想用可视化面板**：可选 **「宝塔 Linux 面板」** 等模板，但安装 Nginx、MySQL、证书时要在宝塔里操作，**本文下面仍以命令行为主**，需你对照宝塔界面自己点。  
   - **会 Docker**：可选带 **Docker** 的镜像，再用仓库里 `Dockerfile` / `docker-compose` 部署；小白可先跳过。  
4. 选配置：学习/小项目可选 **2 核 2G～4G**，带宽按预算选（有 **3M** 新用户价可留意）。  
5. 地域：选离你用户近的；**大陆机房一般要做备案**（见第三步）。  
6. 买好后在控制台里能看到：**公网 IP**（一串数字，例如 `1.2.3.4`）。**复制保存下来。**

> **注意**：香港/境外机房往往**不用备案**就能绑域名，但价格、速度、合规要求与大陆不同，按你课程/公司要求选。

---

## 第三步：开始备案（大陆服务器几乎必做）

**备案入口在哪？**  
和「添加 DNS 记录」**不是同一个页面**。请打开 [腾讯云控制台](https://console.cloud.tencent.com/)，顶部**搜索框输入「备案」或「ICP」**，进入 **「网站备案」**（或 **ICP 备案**）专区按向导操作；**DNSPod「记录管理」、轻量「域名解析」里不会出现备案表单**，所以看不到是正常的。

**要现在就备案吗？**  
- 服务器在**中国大陆**（例如北京），用**域名**对外提供网站/API 服务，按规定需要完成 **ICP 备案**，宜**尽早**在控制台发起。  
- 备案审核要**一段时间**，可与「装环境、传代码、先拿 IP 测后端」**并行**。  
- **没通过前**：可用 **公网 IP**（如 `http://IP:8000/health`）在服务器上自测；小程序**正式上线**仍要 **HTTPS + 合法域名**，一般要等备案通过并配好证书后更稳。

1. 在腾讯云控制台搜 **「网站备案」** 进入专区。  
2. 按页面提示填：**域名、服务器、主体信息**（个人或企业）、上传资料。  
3. **接入云资源时常见提示**：若轻量/CVM **剩余购买时长不足 3 个月**，需先在 **轻量/CVM 控制台续费**（一般续到 ≥3 个月），再回到备案页点 **「刷新」** 云资源有效期，否则无法提交。  
4. **域名栏**：填**主域名**即可，例如 `qhdzoo.site`（不要带 `http://`、`www` 或 `api`；`api` 子域名通常在解析里配置，备案按页面说明填写）。  
5. 提交后进入审核流程（以页面为准）：常见为 **腾讯云审核**（可能 **1～2 个工作日内** 致电主体负责人核验，请留意 **010** 等来电、勿拦截骚扰电话）→ **提交管局** → **工信部短信核验** → **管局审核**（常见 **1～20 个工作日**）。  
6. **备案没通过前**：可以用 **服务器公网 IP** 先调试（见后文「先用 IP 调试」）；域名对外访问规则以管局与腾讯云提示为准。  
7. **旧网站显示「域名已过期」**（如历史域名 `zjiayi.online`）与**新提交的备案**是不同条目；新单审核期间可并行在服务器上部署新项目。

> 若你买的是**境外/香港**服务器，以腾讯云当前规则为准，很多场景**不做 ICP 备案**，但小程序 **request 合法域名仍要是 HTTPS**，证书要合法。

---

## 第四步：把域名指到你的服务器（解析）

备案**通过后**（或你使用境外机且规则允许绑域名时）：

1. 打开 **DNSPod / 域名解析**（腾讯云域名一般在「DNSPod」里解析）。  
2. 选择你的域名 → **添加记录**：  
   - **主机记录**：`api`（表示 `api.你的域名.com`；也可改成你喜欢的，但要和后面 Nginx、小程序里填的一致）。  
   - **记录类型**：`A`。  
   - **记录值**：填你服务器的 **公网 IP**。  
3. 保存。等几分钟到几小时，全球 DNS 会生效。  
4. 在电脑 `cmd` 里测试：`ping api.你的域名.com`，应显示你的服务器 IP（若 ping 不通，有时是服务器禁 ping，以浏览器能否打开为准）。

---

## 第五步：服务器安全组放行端口

1. 腾讯云控制台 → 你的 **轻量/CVM** → **防火墙** 或 **安全组**。  
2. 放行：**TCP 22**（SSH 登录）、**80**（网页/证书）、**443**（HTTPS）。  
3. **不要**对全世界开放 **3306**（MySQL）、**6379**（Redis）。

---

## 第六步：用电脑登录服务器（第一次）

1. Windows 可装 **PuTTY**，或用 **PowerShell**：  
   `ssh ubuntu@你的服务器公网IP`  
   （若镜像用户是 `root`，用 `root@IP`；以腾讯云给的「登录名」为准。）  
2. 用买服务器时设置的**密码**或**密钥**登录。  
3. 能进入黑底白字界面，说明这步成功。

---

## 第七步：在服务器上装基础软件

在 SSH 里**整段复制执行**（Ubuntu 示例）：

```bash
sudo apt update
sudo apt install -y python3-pip python3-venv nginx git mysql-server
```

MySQL 安装后会提示设 root 密码，**记下来**，后面 `.env` 要用。

---

## 第八步：把项目后端传到服务器

任选一种：

- **Git**：服务器上 `git clone` 你的 Gitee/GitHub 仓库。  
- **压缩包**：本地把 `LittleApp` 打成 zip，用 **WinSCP** 或腾讯云控制台「上传」到服务器，再 `unzip`。

建议路径示例：`/home/ubuntu/qhd/LittleApp`（下面命令按此写，你若路径不同要自己替换）。

---

## 第九步：建数据库并导入表结构

**Ubuntu 上常见情况：** 刚装的 MySQL，`root` 往往**没有**你设的密码，而是用 **`auth_socket`**，直接执行 **`mysql -u root -p` 会报错**。请先 **`sudo mysql`** 进入（无需密码），再在 `mysql>` 里执行 **`ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY '你的新密码';`** 和 **`FLUSH PRIVILEGES;`**，之后才能用 **`mysql -u root -p`**；或单独 **`CREATE USER`** 给业务库用（见上文对话说明）。

1. 登录 MySQL：先试 **`sudo mysql`**；若已给 root 设过密码再用 **`mysql -u root -p`**。  
2. 执行（把路径换成你实际上传后的 `init.sql` 路径）：

```bash
mysql -u root -p < /home/ubuntu/qhd/LittleApp/sql/init.sql
```

3. 再登录 MySQL 检查：`SHOW DATABASES;` 里应有 **`qhd_zoo`**。

---

## 第十步：配置后端环境变量

1. 在服务器上：  
   `cd /home/ubuntu/qhd/LittleApp/backend`  
2. 复制示例：  
   `cp .env.example .env`  
3. 用 `nano .env` 编辑（不会用 nano 可搜「nano 保存退出」），**至少改这些**：

| 变量 | 写什么 |
|------|--------|
| `USE_SQLITE` | `false` |
| `MYSQL_PASSWORD` | 你的 MySQL root 密码 |
| `MYSQL_USER` | 一般用 `root` 或你单独建的用户 |
| `MYSQL_DB` | `qhd_zoo` |
| `WX_APPID` | 微信公众平台 → 开发 → 开发管理 → **AppID** |
| `WX_SECRET` | 同上页的 **AppSecret**（生成后只显示一次，要保存） |
| `MOCK_WX_LOGIN` | `false` |
| `JWT_SECRET` | 随便一长串英文数字，**别用默认示例** |

保存退出。

---

## 第十一步：安装 Python 依赖并试跑

```bash
cd /home/ubuntu/qhd/LittleApp/backend
python3 -m venv .venv
source .venv/bin/activate
pip install -U pip
pip install -r requirements.txt
```

试跑（先别关窗口，测完再 Ctrl+C）：

```bash
uvicorn main:app --host 0.0.0.0 --port 8000
```

另开一个 SSH 窗口或在服务器上：

```bash
curl http://127.0.0.1:8000/health
```

若返回带 `"status":"up"` 的 JSON，说明后端 OK。

---

## 第十二步：用 systemd 让后端一直运行

按 **`docs/腾讯云部署指南.md`** 里的 **`qhd-api.service`** 一节，把里面的路径改成你的真实路径（例如 `/home/ubuntu/qhd/LittleApp/backend`），然后：

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now qhd-api
sudo systemctl status qhd-api
```

---

## 第十三步：申请 SSL 证书（HTTPS，小程序必填）

小程序 **request 合法域名必须是 HTTPS**。

1. 腾讯云控制台 → **SSL 证书** → 申请**免费 DV 证书**（按向导绑定域名 `api.你的域名.com`）。  
2. 审核通过后，下载 **Nginx 格式**，上传到服务器，例如：  
   `/etc/nginx/ssl/fullchain.pem`  
   `/etc/nginx/ssl/privkey.pem`  
3. 按 **`docs/腾讯云部署指南.md`** 里 **「六、Nginx 反向代理 + HTTPS」** 编辑 Nginx，把 **`api.yourdomain.com`** 改成你的 **`api.你的域名.com`**，证书路径改成你上传的真实路径。  
4. `sudo nginx -t && sudo systemctl reload nginx`  
5. 浏览器打开：`https://api.你的域名.com/health`  
   应看到 JSON，且地址栏有小锁。

---

## 第十四步：微信公众平台配置 + 改小程序

1. 登录 [微信公众平台](https://mp.weixin.qq.com/) → **开发 → 开发管理 → 服务器域名**。  
2. **request 合法域名** 填：`api.你的域名.com`（**不要** `https://`，**不要**路径）。  
3. 打开你电脑上的小程序项目 **`app.js`**，把 **`apiBase`** 改成：  
   `https://api.你的域名.com`  
4. 微信开发者工具 **上传代码** → 公众平台 **提交审核** → **发布**。

---

## 你现在「刚买完域名」立刻能做的

| 顺序 | 做什么 |
|------|--------|
| 1 | 确认域名在腾讯云控制台里能看见 |
| 2 | **买轻量/CVM**，记下 **公网 IP** |
| 3 | **开始备案**（大陆机）—— 和后面步骤可并行 |
| 4 | 备案通过前：用 **IP** 在服务器上按第七～十二步先把 **后端跑通**（小程序正式环境仍要 HTTPS 域名） |
| 5 | 备案通过后：**第四步解析** → **第十三步 HTTPS** → **第十四步小程序** |

---

## 卡住时怎么查

- **连不上服务器**：检查安全组是否放行 22；IP 是否输对。  
- **域名打不开**：解析是否指向当前 IP；备案是否已通过；Nginx、证书是否配置对。  
- **小程序报域名错误**：合法域名是否**完全一致**；是否 **HTTPS**；`app.js` 的 `apiBase` 是否没多写 `/health`。  
- 更细的命令与配置片段：**`docs/腾讯云部署指南.md`**  
- 本地与真机调试：**`DEBUG.md`**
