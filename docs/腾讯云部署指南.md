# 腾讯云部署指南（小程序 + FastAPI）

**刚买完域名、第一次部署？** 请先按 **[腾讯云小白部署步骤.md](./腾讯云小白部署步骤.md)** 从第一步做到最后；本文提供 **Nginx、systemd、.env** 等配置片段，可与小白文档对照使用。

适用于：在腾讯云 **云服务器 CVM**（或 **轻量应用服务器 Lighthouse**）上部署本项目 **backend**，前端仍在微信开发者工具上传，**API 使用 HTTPS 域名**供小程序 `request` 合法域名配置。

---

## 一、上线前准备

| 项目 | 说明 |
|------|------|
| 腾讯云账号 | 已实名 |
| 服务器 | CVM 或 Lighthouse，建议 **2 核 4G** 起；系统可选 **Ubuntu 22.04** |
| 域名 | 已解析到服务器公网 IP（**A 记录**，如 `api.yourdomain.com`） |
| 备案 | 域名若解析到**中国大陆**服务器，需完成 **ICP 备案**（按腾讯云流程） |
| 小程序 | 微信公众平台配置 **request 合法域名** 为上述域名（仅填主机名，不含 `https://`） |

安全组放行：**22**（SSH）、**80**（HTTP 证书验证/跳转）、**443**（HTTPS）。**不要**对公网开放 **3306**（MySQL）、**6379**（Redis）。

---

## 二、推荐架构

```
用户微信 → HTTPS → 域名 (443) → Nginx → 127.0.0.1:8000 (Uvicorn/FastAPI)
                                    ↓
                          MySQL（本机或 腾讯云数据库 TencentDB）
                          Redis（本机或 腾讯云 Redis，可选）
```

数据库可选：

- **轻量**：MySQL 装在**同一台** CVM，仅监听 `127.0.0.1`。
- **生产**：**TencentDB for MySQL** + **TencentDB for Redis**，在 `backend/.env` 填内网地址与账号。

## 三、服务器初始化（Ubuntu 示例）

SSH 登录后：

```bash
sudo apt update && sudo apt install -y python3-pip python3-venv nginx git
# MySQL 若本机安装（示例）
sudo apt install -y mysql-server
```

创建运行用户与目录（示例 `/opt/qhd-api`）：

```bash
sudo mkdir -p /opt/qhd-api && sudo chown $USER:$USER /opt/qhd-api
```

将 **`LittleApp/backend`** 整目录上传到服务器（`scp`、`git clone`、或控制台上传均可），例如：

```text
/opt/qhd-api/backend/
  main.py
  app/
  requirements.txt
  ...
```

## 四、Python 环境与依赖

```bash
cd /opt/qhd-api/backend
python3 -m venv .venv
source .venv/bin/activate
pip install -U pip
pip install -r requirements.txt
```

在服务器创建 **`/opt/qhd-api/backend/.env`**（**不要**提交到 Git），可参考仓库内 `.env.example`，生产注意：

```env
USE_SQLITE=false
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_USER=你的用户
MYSQL_PASSWORD=强密码
MYSQL_DB=qhd_zoo

WX_APPID=小程序AppID
WX_SECRET=小程序AppSecret
MOCK_WX_LOGIN=false

JWT_SECRET=足够长的随机串

REDIS_HOST=127.0.0.1
REDIS_PORT=6379
# 若不用 Redis，可先保持默认；登录写 Redis 失败会降级

ALLOW_MOCK_LLM=false
# 若接大模型再填 LLM_API_BASE / LLM_API_KEY / LLM_MODEL
```

在 MySQL 中执行建表（将本地 `sql/init.sql` 上传到服务器后）：

```bash
mysql -u root -p < /opt/qhd-api/sql/init.sql
```

（路径按你实际上传位置调整。）

## 五、使用 systemd 常驻 Uvicorn

```bash
sudo nano /etc/systemd/system/qhd-api.service
```

示例内容（注意 **`User`**、**路径**、**WorkingDirectory** 按实际修改）：

```ini
[Unit]
Description=QHD Zoo FastAPI
After=network.target mysql.service

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/opt/qhd-api/backend
Environment="PATH=/opt/qhd-api/backend/.venv/bin"
ExecStart=/opt/qhd-api/backend/.venv/bin/uvicorn main:app --host 127.0.0.1 --port 8000
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now qhd-api
sudo systemctl status qhd-api
```

本机自检：`curl -s http://127.0.0.1:8000/health`

## 六、Nginx 反向代理 + HTTPS

### 1. 申请证书（任选其一）

- **腾讯云 SSL 证书**（控制台申请免费 DV，下载 Nginx 格式，上传到服务器）。
- 或使用 **Let's Encrypt**（`certbot`）自动续期。

假设证书路径：

- 证书：`/etc/nginx/ssl/fullchain.pem`
- 私钥：`/etc/nginx/ssl/privkey.pem`

### 2. Nginx 站点配置

```bash
sudo nano /etc/nginx/sites-available/qhd-api
```

示例（将 `api.yourdomain.com` 换成你的域名）：

```nginx
server {
    listen 80;
    server_name api.yourdomain.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name api.yourdomain.com;

    ssl_certificate     /etc/nginx/ssl/fullchain.pem;
    ssl_certificate_key /etc/nginx/ssl/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用并重载：

```bash
sudo ln -sf /etc/nginx/sites-available/qhd-api /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
```

浏览器访问：`https://api.yourdomain.com/health` 应返回 JSON。

## 七、微信小程序侧

1. **微信公众平台 → 开发 → 开发管理 → 服务器域名 → request 合法域名**  
   填写：`api.yourdomain.com`（与证书域名一致）。

2. 修改本项目 **`app.js`**：

```js
globalData: {
  apiBase: "https://api.yourdomain.com",
  ...
}
```

3. 微信开发者工具 **上传** → 公众平台 **提交审核** → **发布**。

## 八、可选：腾讯云托管形态

| 形态 | 说明 |
|------|------|
| **轻量 + 单机** | 本指南默认；MySQL/Redis 同机或买 TencentDB。 |
| **容器服务 TKE** | 将 `Dockerfile` 构建镜像部署；需自行配置 Ingress/CLB 与 HTTPS。 |
| **云托管 CloudBase Run** | 适合容器化 API；需按平台要求改监听端口与环境变量。 |

仓库根目录 **`docker-compose.yml`** 可在服务器安装 Docker 后用于快速起 **MySQL + Redis + API**（需把 API 镜像或 build 指向本仓库，并配置环境变量与公网入口）。

## 九、检查清单

- [ ] 域名 HTTPS 可访问 `/health`、`/health/db`（`db` 为 `mysql` 且成功）  
- [ ] `.env` 已关 `MOCK_WX_LOGIN`，微信密钥为正式  
- [ ] `JWT_SECRET` 已更换  
- [ ] 安全组未暴露数据库端口  
- [ ] 小程序 `apiBase` 与合法域名一致  
- [ ] 备份策略（数据库定期快照/导出）
