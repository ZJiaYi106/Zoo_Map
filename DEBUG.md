# 项目调试说明

## 一、微信开发者工具

1. **编译失败 / 页面空白**
   - 确认 `app.json` 中 `pages` 路径与实际文件一致。
   - 关闭「ES6 转 ES5」等冲突项后重试（若基础库较新，一般保持默认即可）。
2. **请求失败、提示网络异常**
   - 勾选 **不校验合法域名**。
   - 确认 `app.js` 里 `apiBase` 与后端一致；真机不要用 `127.0.0.1`，应改为电脑局域网 IP。
3. **地图不显示 / 无定位**
   - 在开发者工具中开启模拟定位；真机需授权位置权限。
   - 检查 `app.json` 中 `permission` 与 `requiredPrivateInfos`。
4. **登录失败**
   - 本地开发：后端设置 `MOCK_WX_LOGIN=true`，无需真实 `AppSecret`。
   - 正式环境：配置 `WX_APPID`、`WX_SECRET`，并保证 `wx.login` 的 code 在服务端换取 openid 成功。

### 真机提示 `request:fail url not in domain list`（含 `:10.x.x.x` 等 IP）

这是微信客户端在**真机**上拦截请求；与电脑浏览器能打开 `/health` **不矛盾**（浏览器不做小程序域名校验）。

请按顺序做（多数人是第 2、3 步）：

1. **项目配置**：仓库里 `project.config.json` 已含 `"urlCheck": false`；改完后在开发者工具点 **「编译」** 一次。  
2. **不要用普通「预览」扫码**：请点工具栏 **「真机调试」**，用手机微信扫**真机调试**的码进入小程序，再测「测试连接 / 登录」。  
3. **手机端打开调试**：进入小程序后点右上角 **「···」**，找到 **「打开调试」/「开发调试」**（不同版本文案略有差异）并开启，再重试请求。  
4. **先关 VPN**：截图里若开了 **VPN**，常导致访问不了局域网 IP，试玩时先关掉 VPN。  
5. 仍失败时：用 **内网穿透**（如 ngrok、cpolar）得到 **HTTPS 域名**，在公众平台加到 **request 合法域名**（正式环境方案）；本地开发优先用上面 2、3 步。

6. **详情 → 本地设置**里仍建议勾选 **「不校验合法域名…」**，与 `urlCheck:false` 一致；每次修改后 **重新编译** 再扫码。  
7. 若手机**浏览器**能打开 `/health`，小程序仍不行：多半是第 2～3 步未做；若**浏览器也打不开**，再查防火墙、同网段、AP 隔离。上线正式版需在公众平台配置 **HTTPS 合法域名**。

### 状态码 404

- **接口根地址**应只写到端口：`http://电脑IP:8000`，**不要**带 `/health`、`/api`（否则会请求成 `/health/health` 等，后端无此路由即 404）。保存后代码会自动去掉误填的路径。
- 浏览器访问 `http://IP:8000/` 应看到 JSON（含 `service`）；若没有，可能连错了端口或不是本项目的 uvicorn。

### 浏览器能打开 /health，但微信登录报 HTTP 500

`/health` **不访问数据库**；`/api/auth/wx-login` 要 **读写用户表**。常见原因是 **MySQL 未启动或未建表**。

- **推荐本地调试**：在 `backend/.env` 保持 **`USE_SQLITE=true`**（代码默认已是 true），重启 `uvicorn` 后会自动在 `backend/data/dev.db` 建表，无需单独装 MySQL。
- 若使用 MySQL：先启动 MySQL，再执行 `sql/init.sql`，并设 **`USE_SQLITE=false`** 且填写正确账号密码。
- 自检：浏览器访问 **`http://你的IP:8000/health/db`**，应返回 `"ok"` 与 `"db":"sqlite"` 或 `"mysql"`。

### 电脑连「手机热点」时怎么填接口地址

热点场景下：**手机开热点，电脑连该 WiFi**，此时电脑与手机在同一网段，应访问 **电脑在热点网段里的 IPv4**，不能用 `127.0.0.1`。

1. 在 **电脑**上打开 `cmd`，执行：`ipconfig`。
2. 找到当前上网用的那个适配器（名称常含 **WLAN**、**无线** 或 **本地连接**），看 **IPv4 地址**。
   - Android 热点常见：`192.168.43.x`、`192.168.137.x` 等。
   - iPhone 热点常见：`172.20.10.x` 等。
3. 在小程序登录页「接口地址」填：`http://你的IPv4:8000`（示例：`http://192.168.43.156:8000`），点 **保存地址** → **测试连接**。
4. 后端必须监听所有网卡：`uvicorn main:app --host 0.0.0.0 --port 8000`；Windows 防火墙需放行 **入站 TCP 8000**。

## 二、后端接口调试

1. 浏览器或 curl 访问：`GET http://127.0.0.1:8000/health`。
2. **真机联调**：手机上的 `127.0.0.1` 是手机自己。请在登录页填写电脑的 **局域网 IP**（见上一节），并点「测试连接」。后端需 **`--host 0.0.0.0`**，并放行防火墙。
3. 使用 **Postman / Apifox**：`POST /api/auth/wx-login`，Header 带 `Authorization: Bearer <token>` 测其它接口。
4. **MySQL / Redis 连接失败**：检查 `backend/.env`；Docker 下 MySQL 主机名为 `mysql`。

## 三、AI 功能调试

1. **无 API Key**：保持 `ALLOW_MOCK_LLM=true`，接口返回内置内容。
2. **有 API Key**：配置 `LLM_API_BASE`、`LLM_API_KEY`、`LLM_MODEL`。
3. **知识库**：编辑 `backend/app/services/knowledge_base.py` 观察 RAG 命中效果。

## 四、常见问题

| 现象 | 处理 |
|------|------|
| `url not in domain list` | 见上文「真机提示」；勾选不校验域名后重新编译/预览或改用真机调试 |
| 401 未登录 | 先登录；`utils/request.js` 会清理 token |
| 景点列表为空 | 执行 `sql/init.sql` |
| 语音无声音 | 需后端返回可访问的 `tts_url` 或使用系统读屏 |
